# Travel-eats 프로젝트 개선 계획 (프론트엔드 중심)

## 높은 우선순위 (핵심 기능)

- **[완료] 모의(Mock) 기능 구현 (localStorage 활용)**:
  - 인증: 실제 API 호출 없이 로그인/회원가입 및 세션 관리 기능 구현
  - 그룹: 그룹 생성, 참여, 정보 조회, 멤버 관리 기능 구현
  - 데이터 관리: 여행 계획, 음식 선호도 등 사용자 데이터를 `localStorage`에 저장 및 조회
- **[완료] Google Places API 연동**:
  - 음식점 검색 및 상세 정보 조회를 위한 API 호출 함수 구현
  - 기존 추천 프로세스(`FoodPreferencePage` -> `FoodResultPage`)에 Places API 연동
  - API 키를 `.env.local` 파일로 분리 및 `gitignore`에 추가하여 관리 (보안 강화)
  - [완료] 지도 기반 여행 계획 기능 구현 (일자별 위치 및 반경 설정)
  - [완료] 일자별 추천 프로세스 연동 (Nearby Search API 활용)
- **[보류] 전역 상태관리 도입**:
  - `Zustand` 또는 `Redux Toolkit`을 사용하여 `localStorage`의 데이터를 포함한 앱 전반의 상태(그룹 정보, 사용자 세션, 추천 결과 등)를 효율적으로 관리
